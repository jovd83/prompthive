# Refactoring Roadmap: prompt-hive

## Executive Summary
**Maintainability Score: 68/100**

**State of the Union:** 
The codebase has seen significant architectural improvements with the recent extraction of business logic into distinct service layers and the establishment of global constants to eliminate magic values. However, residual technical debt persists, primarily in the form of suppressed exception handling (silent failures) and localized type safety erosion that still threaten production reliability and scaling.

## The Audit Table

| Location | Severity | Smell Detected | The Risk | The Fix |
| :--- | :--- | :--- | :--- | :--- |
| `services/imports.ts` (~800 lines) | High | God Object | File handles payload parsing, normalization of legacy formats, filesystem attachments processing, and complex nested database insertions. Modifying import structure risks cascading failures in other areas. | Decompose the module into distinct services: `ImportParserService`, `LegacyFormatMapper`, and `ImportTransactionService`. |
| `services/imports.ts:22,54,85` <br> `services/files.ts:25,41` | High | The "Silent Failure" Trap | Catch blocks suppress thrown exceptions without logging or propagating the error. Failures during file reading, generation, or parsing result in silent data loss or phantom state. | Implement structured logging (`console.error` with context) and propagate errors (or fallback states) so orchestrators can handle them. |
| `components/collection-view/CollectionSidebar.tsx:97,140,530` | Medium | Ghost State / Silent Failures | UI state operations (fetching nodes, expanding folders) fail silently in empty try/catch blocks. When the backend or network fails, the user is left with a broken UI that appears "stuck" without any actionable feedback. | Integrate structured error state management and surface user-facing alerts (e.g., toast notifications) when CRUD actions fail. |
| `services/user.ts:78,90` | Low | Type Safety Erosion | Validating and saving roles using `as any` casts (`Object.values(ROLES).includes(role as any)`) bypasses the Typescript compiler's strict type checking on critical authorization fields. | Cast the literal value properly using a Type Guard (`return role in ROLES;`) or parse it safely through Zod Enum schemas before interacting with the database layer. |
| `lib/export-client.ts:52` <br> `components/ImportExportContent.tsx:57` | Low | Type Safety Erosion | Heavy usage of `catch (e: any)` or casting unknown types as `any` in client-side orchestration. Defeats TS strict mode configurations. | Switch to `catch (e: unknown)` and explicitly map to string using `e instanceof Error ? e.message : String(e)`. |

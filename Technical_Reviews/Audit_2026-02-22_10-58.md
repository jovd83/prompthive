# Refactoring Roadmap: prompt-hive (Round 3)

## Executive Summary
**Maintainability Score: 76/100**

**State of the Union:** 
The boundary between the data layer (Services/Actions) and the presentation layer (Components) is thoroughly secured—no raw database queries or direct `prisma` imports exist on the client side. The primary backend services have been cleanly factored. However, the UI layer itself is now the primary technical debt hotspot, suffering from a heavy reliance on `as any` casting for complex prop drilling and the emergence of "God Components."

## The Audit Table

| Location | Severity | Smell Detected | The Risk | The Fix |
| :--- | :--- | :--- | :--- | :--- |
| `components/collection-view/CollectionSidebar.tsx` (~600 lines) | High | God Component | Extremely high cognitive load. This massive file mixes drag-and-drop orchestration, bulk database mutations, complex local state (progress tracking, modals), and UI rendering. Any change risks breaking unrelated sidebar features. | Decompose the file. Extract `BulkTagModal` and `CollectionHeader` into separate component files. Extract the mutation logic and local state into a custom hook `useCollectionSidebar()`. |
| `app/(dashboard)/*` &<br>`components/*` | Medium | Type Safety Erosion (`as any`) | Deeply nested Prisma includes (e.g., `Collection` with `Prompts` and `Tags`) are repeatedly passed as `collection as any` or `settings as any` into UI components. This totally subverts the typescript compiler and risks runtime crashes if the schema evolves. | Define strict, decoupled UI interfaces (e.g., `export interface UICollection { ... }`) in a central type definitions file. Map the Prisma outputs to these strict interfaces *before* passing them as props. |
| UI Forms & Handlers (`components/**/*.tsx`) | Medium | Implicit `any` in Try/Catch | Dozens of `catch (error) { ... }` blocks exist in local UI handlers. Because TS defaults to `any` here, it often suppresses the actual error payload or results in generic "Failed to save" toasts that are impossible to debug for users. | Enforce "strict type checking" in tsconfig or globally switch to `catch (error: unknown)` and aggressively map them: `const msg = error instanceof Error ? error.message : String(error);`. |
| `actions/prompts.ts` (~280 lines) | Low | Bloated Action Router | Centralizes over 15 distinct Server Actions covering creation, bulk edits, linking, versioning, and tagging. It is becoming a bottleneck for imports and orchestration. | Partition the module into functional silos: `actions/prompt-crud.ts`, `actions/prompt-bulk.ts`, and `actions/prompt-links.ts`. |
